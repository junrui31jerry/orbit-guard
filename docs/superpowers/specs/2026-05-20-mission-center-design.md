# OrbitGuard 任務中心設計文件

日期：2026-05-20
狀態：已同意進入規劃

## 目標

在 OrbitGuard 中新增一個乾淨、任務導向的系統中心，讓使用者不用記住所有快捷鍵，也能完成一套完整的展示流程。

這個互動體驗要讓使用者清楚知道：

- 目前進行到哪一個步驟。
- 哪些步驟已經完成。
- 現在最適合做的下一個動作是什麼。
- 發射、風險檢視、避碰處置、儲存報告後，各自造成了什麼變化。

## 目前專案狀態

OrbitGuard 是一個使用 C++17 與 raylib 製作的 3D 碰撞風險監控原型。目前系統已經支援：

- 使用 `Tab` 選擇軌道參數。
- 使用 `Left` 與 `Right` 調整軌道參數。
- 使用 `L` 發射使用者衛星。
- 使用 `A` 執行避碰處置。
- 使用 `S` 儲存風險報告。
- 暫停、重置、速度調整、攝影機移動，以及發射操作說明。

目前主要的使用性問題是：這些功能大多以快捷鍵形式存在。右側面板雖然已經顯示許多有用資料，但還沒有引導使用者完成一個完整任務。

## 設計方向

在現有右側系統中心加入任務清單。不要新增複雜分頁、過多彈窗，或大型教學覆蓋層作為主要體驗。

右側面板維持整潔，並由上到下分成四個區塊：

- 系統狀態
- 任務流程
- 風險洞察
- 下一步提示

現有的發射操作說明覆蓋層可以保留，作為較完整的說明畫面。任務中心本身只顯示短而重要的提示。

## 任務流程

任務共分成五個步驟：

1. 設定軌道
2. 發射衛星
3. 檢視風險
4. 執行避碰
5. 儲存任務報告

每個步驟都要顯示簡潔的狀態：

- 已完成
- 目前步驟
- 等待中

目前步驟要比等待中的步驟更醒目。已完成的步驟仍要保留在畫面上，但視覺上可以稍微降低強度，讓使用者看得出進度，又不會讓面板變得混亂。

## 互動行為

程式啟動時，任務從「設定軌道」開始。下一步提示應該引導使用者使用 `Tab`、`Left`、`Right` 調整發射參數。

使用者按下 `L` 之後，「發射衛星」變成已完成，任務進入「檢視風險」。

當模擬分析完使用者發射的衛星，並顯示目前風險狀態後，「檢視風險」就視為完成。如果風險是低風險，使用者可以繼續觀察，或直接儲存任務報告。如果風險是中風險或高風險，而且系統有可用的避碰方案，下一步提示應該引導使用者按下 `A` 執行避碰。

使用者按下 `A` 並成功套用避碰方案後，「執行避碰」變成已完成。面板應該顯示現有避碰預覽中的避碰前後距離。

使用者按下 `S` 後，「儲存任務報告」變成已完成。最後訊息應該確認報告已儲存到 `orbit_guard_report.txt`。

## 提示文字

任務中心一次只顯示一個主要的下一步提示。範例：

- 下一步：調整軌道設定，然後按 L 發射。
- 下一步：檢視最近物件組合的風險等級。
- 下一步：按 A 套用系統建議的避碰動作。
- 下一步：目前風險偏低，可以在準備好後儲存任務報告。
- 任務完成：報告已儲存到 orbit_guard_report.txt。

特殊情況要使用友善的提示：

- 如果使用者還沒發射衛星就按 `A`，提示他們先發射使用者衛星。
- 如果目前風險偏低，不要讓避碰看起來像是必要操作。
- 如果沒有任何快速避碰方案能改善狀況，提示使用者調整軌道設定後重新發射。

## 資料模型

新增一個小型任務狀態模型，避免任務進度判斷分散在 UI 繪製邏輯裡。

建議狀態包含：

- `hasLaunchedUserSatellite`
- `hasReviewedRisk`
- `hasAppliedAvoidance`
- `hasSavedMissionReport`
- `currentStep`
- `nextActionText`

任務狀態應該在 `main.cpp` 中根據既有使用者操作更新，再傳入 UI 繪製層。

## 畫面呈現

UI 要保持緊湊。任務流程應該放進現有右側面板，不要遮住 3D 模擬畫面。

面板要使用簡短標籤與清楚層級：

- 小標題：`Mission Flow` 或 `任務流程`
- 五個緊湊的步驟列
- 一行簡短的 `Next Action` 或可換行的下一步提示

不要把大量教學文字放進系統中心。較長的操作說明應該留在按 `H` 顯示的發射說明覆蓋層。

## 錯誤處理

不需要新增阻擋式錯誤畫面。使用現有的動作訊息區與下一步提示提供回饋即可。

如果使用者沒有照順序操作，程式不應該卡住。任務狀態仍然要保持容易理解，並引導使用者回到合理的流程。

## 測試方式

這個原型使用手動驗證即可：

- 啟動程式。
- 確認任務從「設定軌道」開始。
- 調整發射設定並發射一顆使用者衛星。
- 確認「發射衛星」與「檢視風險」會正確更新。
- 在有可用避碰方案時按下 `A`，確認「執行避碰」會正確更新。
- 按下 `S`，確認「儲存任務報告」完成。
- 確認 `orbit_guard_report.txt` 仍然會產生。
- 按下 `R`，確認任務狀態會清楚重置。

## 不包含的範圍

這次變更不新增滑鼠點擊按鈕、任務歷史紀錄、多種任務模式，或獨立的報告檢視器。這些可以等核心引導流程穩定後，作為後續功能再擴充。
